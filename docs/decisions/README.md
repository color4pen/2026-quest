# Architecture Decision Records (ADR)

技術的な意思決定の記録。「何を決めたか」だけでなく「なぜそうしたか」「他に何を検討したか」を残す。

## 運用ルール

- ファイル名: `NNN-短い説明.md`（例: `001-domain-layer-dependency.md`）
- 新規追加時は番号を連番で振る
- ステータス: `提案中` → `承認済み` → `廃止`（必要に応じて）

## 記録すべき判断の基準

「後から誰かに"なぜこうしたの？"と聞かれそうな判断」に絞る。

例:
- アーキテクチャの層構造
- 設計パターンの採用理由
- ライブラリ選定の根拠
- パフォーマンスとシンプルさのトレードオフ
