# タスク実行ワークフロー

以下の手順でタスクを実施してください。
リファクタリング、機能追加、コンテンツ実装など全般に対応。

## Step 1: ブランチ作成

`feature/{タスク名}` ブランチを作成:

```bash
git checkout -b feature/{タスク名}
```

## Step 2: 設計ドキュメント作成

`docs/changes/YYYYMMDD-{タスク名}/design.md` を作成:

```markdown
# {タスクタイトル}

## 種別
- [ ] リファクタリング
- [ ] 機能追加
- [ ] コンテンツ実装
- [ ] バグ修正
- [ ] その他

## 概要
何を、なぜ変更/追加するか

## 現状分析
- 関連ファイル・既存実装の調査
- 変更が必要な箇所の特定

## 設計方針
- 新規ファイル/クラス/関数
- 既存コードへの変更
- データ構造/インターフェース

## 実装計画
- Step 1: ...
- Step 2: ...
- Step 3: ...

## 影響範囲
- 変更されるファイル一覧
- テストへの影響

## 承認
- [ ] 設計レビュー完了
- [ ] 実装開始
```

## Step 3: ユーザー承認

設計ドキュメントを提示し、承認を得てから実装開始。

## Step 4: 実装

- TodoWriteでタスク管理
- 小さな単位で変更
- 各ステップ後に型チェック (`npx tsc --noEmit`)

## Step 5: テスト

- 必要に応じてテスト追加
- `npm test` で全テスト通過確認

## Step 6: コミット・PR

- 変更単位でコミット
- design.mdは参照用に残す
- PRを作成してレビュー依頼
